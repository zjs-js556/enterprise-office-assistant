# 比赛演示讲解稿

> 建议演示时长：5-8 分钟
> 演示方式：后端终端 + 移动端模拟器 分屏展示

## 开场（30 秒）

各位评委老师好，我演示的项目是「企业办公助手」—— 一个面向中小企业的内部移动办公 App。

它解决了什么问题呢？很多小公司还在用 Excel 管理员工和设备资产，信息分散、容易出错。这个 App 把员工管理、设备资产、分类目录整合在一个移动端工具里，扫码就能用。

技术栈方面，前端是 React Native + Expo，一套代码同时跑 iOS 和 Android。后端是 Python Flask，数据库 MySQL，认证用 JWT。

下面我来演示核心功能。

---

## 第一幕：后端架构展示（1 分钟）

**【切换终端，展示后端代码结构】**

后端采用标准的分层架构：routes 路由层 → services 业务逻辑层 → models 数据模型层 → schemas 校验层。

先看几个关键设计：

**统一响应格式** — 所有接口返回 `{ code, message, data }` 三段式结构，前端不需要判断各种格式。

**全局异常处理** — 400 参数错误、401 未授权、404 不存在、409 业务冲突、422 字段校验失败，全部统一拦截，不需要在每个接口里写 try-catch。

**请求日志** — 每次 API 请求自动记录 IP、方法、路径、状态码、耗时，输出到控制台和日志文件，方便排查问题。

**【启动后端服务】**
```bash
python run.py
```

服务已启动在 5000 端口，Swagger 文档自动生成。

---

## 第二幕：登录认证（1 分钟）

**【切换模拟器，展示登录页】**

打开 App，首先看到登录页。配色是企业在蓝色调，顶部是品牌区，下方白色卡片是登录表单。

输入默认管理员账号 admin / admin123456，点击登录。

**【登录成功，进入首页】**

登录成功后，JWT token 保存在手机本地。后续每个请求的请求头自动带上 `Authorization: Bearer token`，用户无感知。

首页展示用户名和角色，三个功能入口：员工管理、设备分类、设备管理。

**【演示 token 过期处理】**

如果 token 过期了，后端返回 401，前端 axios 拦截器会自动清除本地 token 并跳回登录页，不需要用户手动处理。

## 第三幕：员工管理 CRUD（1.5 分钟）

**【点击"员工管理"】**

进入员工列表。目前是空的，点击右下角的加号按钮新增员工。

**【新增员工表单】**

输入姓名"张三"，年龄 28，邮箱 zhangsan@company.com。

这里有前端校验：如果姓名空着、年龄填 17（低于 18）、邮箱格式不对，都会在对应字段下面显示红色提示，不会发起请求。

提交成功，自动返回列表，下拉可以刷新。

**【查看详情】**

点击"张三"进入详情页，展示员工头像、基本信息和创建时间。

**【编辑员工】**

点击编辑按钮，修改姓名为"张三丰"，年龄改为 30，保存。返回详情页已经更新。

**【删除确认】**

回到列表页，长按员工卡片弹出删除确认框——"确定要删除张三丰吗？此操作不可撤销。"点击确认删除，列表更新。

---

## 第四幕：设备分类管理（1 分钟）

**【点击底部 Tab"分类"】**

进入设备分类列表。新增两个分类："笔记本电脑"和"显示器"。

注意到每个分类右侧显示设备数量，目前都是 0。

点击分类可以编辑名称，长按可以删除。但如果分类下有设备，删除时后端会校验并返回 409：**"分类下存在设备，无法删除"**。这是业务逻辑的保护。

---

## 第五幕：设备管理 + 关联查询（1.5 分钟）

**【点击底部 Tab"设备"】**

进入设备列表。顶部是分类筛选栏——"全部"、以及刚才创建的分类。这些筛选标签点击可以切换，也可以取消。

新增一台设备：名称"ThinkPad X1"，型号"Carbon Gen11"，选择分类"笔记本电脑"。

这里的分类选择器是自定义的下拉列表，不是系统原生 Picker，方便统一视觉风格。

再新增一台 MacBook Pro，同样选笔记本电脑分类。

**【演示分类筛选】**

在筛选栏点击"笔记本电脑"，列表只显示 2 台笔记本。点击"全部"恢复。

**【演示关联查询】**

回到分类列表——"笔记本电脑"右侧已经显示 device_count 为 2。这个数字是后端通过 SQLAlchemy ORM 的 `category.devices.count()` 实时计算的。

**【演示删除保护】**

尝试删除"笔记本电脑"分类 —— 弹出错误："分类下存在设备，无法删除"。必须先删除设备才能删除分类。

**【完整删除流程】**

切换到设备列表，长按删除 ThinkPad X1 → 确认。再回到分类列表，这次删除成功。

---

## 总结（30 秒）

以上就是企业办公助手的全部核心功能。总结一下亮点：

1. **前后端完全分离** — RESTful API + 跨平台移动端
2. **JWT 无状态认证** — token 自动管理，401 自动跳转
3. **分层架构** — routes/services/models/schemas，职责清晰易扩展
4. **完整的异常处理** — 7 种 HTTP 异常 + 字段级校验错误
5. **外键关联保护** — 分类下有设备时禁止删除，数据一致性有保障
6. **全量请求日志** — 每个请求的方法、路径、状态码、耗时全记录
